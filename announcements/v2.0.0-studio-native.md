# GodeX Studio v2.0.0 (Tauri 2 原生窗口)

把 v1.0.3 那个 Bun + WebView2 手撸出来的 98MB "伪桌面应用" 重写为 **Tauri 2 原生 Rust + WebView2** 桌面应用。启动后只弹一个原生窗口，没有命令行，没有浏览器标签页，没有 HTTP 端口。

## 这次改了什么

### 整个 `studio/` 目录被 `studio-tauri/` 取代

| | 旧 `studio.exe` (Bun + WebView2 loader) | 新 `godex-studio.exe` (Tauri 2) |
|---|---|---|
| 入口 | bun runtime + 裸 WebView2 loader | Tauri 2 Rust 运行时 |
| 后端 | TypeScript (`Bun.serve`) | Rust (`#[tauri::command]`) |
| 前端 | 同一份 HTML | 同一份 HTML，但 `fetch()` 全替换为 `invoke()` |
| 窗口 | 自己用 WebView2 loader 创 | Tauri 帮你创 |
| 日志 | Bun 子进程 stdout → SSE → `EventSource` | Rust 线程读 stdout → Tauri event → `listen()` |
| 体积 | 98MB（含 bun runtime） | ~10–15MB（Tauri 编译优化后） |
| 启动 | ~500ms | ~200ms |

`studio/` 整个目录在 `godex/studio-tauri` 分支里不再存在，**所有功能平迁到 `studio-tauri/`**。

### 新增的 Rust 模块（`studio-tauri/src-tauri/src/`）

- `state.rs` — Studio 全局状态：当前 godex 进程句柄、最近日志 ring buffer（1000 行）、持久化路径（config.yaml / godex.exe）、`godex_port` (Mutex<Option<u16>>) 用于端口冲突追踪。启动时 `GodexSupervisor` 用默认路径 seed，避免空配置下首次启动失败。
- `godex.rs` — `kill_process_on_port` 辅助 + 进程句柄存储 + 端口字段。spawn godex 时 `stdio: ['ignore','pipe','pipe']`，stdout/stderr 实时进 ring buffer，按 emoji / keyword 分类 level (`info/warn/error/debug`)。
- `commands.rs` — Tauri 命令：
  - `list_providers` / `upsert_provider` / `delete_provider` — 增删改查 `config.yaml` 里的 `providers` 块；
  - `fetch_remote_models` — POST 远程 provider `/v1/models`，把返回的 model id + context_window + 多模态字段回填到 UI；
  - `set_enabled_models` — 写 `models.enabled[]` 到 config.yaml，**不重启 godex**；
  - `restart_godex` — 通过 supervisor kill + spawn，**只重启 godex 进程**，Studio 不退；
  - `check_port` / `kill_pid` / `find_free_port` — 端口冲突检测 + 强制杀占用进程 + 自动找下一个可用端口。
- `config.rs` — YAML 解析 + provider 块原地编辑（`replace_or_insert` / `remove_block` 用 byte offset 不重写整文件）。保留旧 `enabled[]` 解析路径，新增测试覆盖 camelCase 参数透传、`fetch_remote_models` 无 args 报错路径。
- `strip_ansi.rs` — 去 ANSI 转义码，日志面板只显示纯文本。
- `lib.rs` — `withGlobalTauri` 打开，`window.__TAURI__` 注入前端；`tauri_plugin_dialog` 注册，赋予文件选择能力（让 UI 能浏览 `config.yaml` 路径）。

### UI 变更（`studio-tauri/src/index.html`）

- **两栏布局**：左栏 provider 列表（CRUD），右栏 active provider 编辑器。
  - 连接设置：Name / Base URL / API Key / Spec / Timeout / Note；保存按钮放顶部（之前的"滚动到底才能点"被吐槽）。
  - 模型操作区：从远程拉取模型 → 复选框选启用 → 写回 `enabled[]`。来源 provider 全部标注，多 provider 同名模型不混淆。
  - 模型参数（context_window / max_tokens / 多模态）优先从上游拉，拉不到允许手动填，空白不写入 config。
- **日志面板拆成两个独立窗格**：Studio 自身日志 + GodeX 网关日志，默认上下平分，中间有可拖动 sash 调节比例。
- **所有 sash 都可拖**：左/右、上/下、Studio/GodeX 之间的分割条都能拖动改变比例；超视野的内容全部出滚动条。
- **端口冲突弹模态框**：保存 settings 时立即写 `server.port`，重启前 `check_port` 检测占用 → 给出"换端口"或"关闭占用进程（PID + 进程名）"选项。不冲突不弹窗。
- **`preCheckAndRestartGodex`**：启动 godex 之前先做端口体检，发现冲突就引导用户决策，绝不让 godex 闪退。
- **多模态选项拆细**：multimodal 拆成 image_input / audio_input / video_input / image_output / audio_output 五个独立复选框；上游能拉到就自动勾上。

## 构建

```cmd
cd D:\Documents\VibeCoding\GodeX\studio-tauri
cargo tauri build
```

首次 build 5-15 分钟，产出：
- `src-tauri\target\release\godex-studio.exe` (~10-15MB)
- `src-tauri\target\release\bundle\nsis\GodeX Studio_0.1.0_x64-setup.exe` (NSIS 安装包)

## 前置（一次性）

1. **Rust 工具链**：从 https://rustup.rs/ 装 (`rustup-init.exe`)，默认安装 `stable-x86_64-pc-windows-msvc` 即可。
2. **MSVC C++ Build Tools**：Visual Studio Installer 装 "Desktop development with C++"（需要 `cl.exe` 和 `link.exe`）。
3. **WebView2 Runtime**：Win11 自带，Win10 21H2+ 也自带，更早要装 https://developer.microsoft.com/microsoft-edge/webview2/ 。
4. **Tauri CLI**：`cargo install tauri-cli --version "^2.0"`（装好后有 `cargo tauri` 子命令）。

## 与 Codex 的协作模式

启动 `godex-studio.exe` 后它会自己 `spawn` 一份 godex 子进程（端口默认 5678，被占用就引导换），Codex 在另一端继续对话，**整个开发期间不需要重启 Codex 会话**。两个二进制版本交替构建（旧的保留作为回滚，新版跑完验证完再切）以避免打断 Codex 与 godex 的活跃连接。

## 不在这次改动范围内

- `src/` 下的 godex 内核一行没动 —— `godex/studio-tauri` 分支只新增 `studio-tauri/` 目录 + `announcements/`。上游 MiniMax tool_call / context window 等修复在 `codex/fix-minimax-tool-call-arguments` 上保留，本分支不引入。
- `studio/` 旧 Bun 版本整个目录被删除 —— 不会再回滚到 v1.0.3 的 Bun + WebView2 路线。
- `studio-tauri/` 里 `target/` 是 cargo build 产物，git 忽略，不进仓库。
