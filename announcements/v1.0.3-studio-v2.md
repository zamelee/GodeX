# GodeX Studio v2:两栏 UI + models.enabled[] + 内嵌 godex 生命周期

GodeX Studio v1 (v1.0.2) 的 UI 是三栏 (Provider list / 表单 / Model 列表),保存配置后还要手动 restart godex,日志是读 trace DB 慢路径。这次重写改成了两栏布局 (Provider list / Active Provider Editor),把 **`models.enabled[]`** 作为新的配置入口,并把所有 godex 进程管理内嵌到 Studio 一个 `.exe` 里。

## 这次改了什么

### godex 侧 (`src/`)

- `src/config/schema.ts` — 新增 `ModelCapabilities` (8 个 bool: text / image_input / audio_input / video_input / image_output / audio_output / tool_use / stream) + `EnabledModel` + `ModelsConfig.enabled?`。
- `src/config/sections/models.ts` — `parseModelsConfig` 解析 `aliases` + `enabled[]`,把 `enabled[]` 自动展开成内部 alias map (`<provider>/<model> -> <provider>/<model>`),用户显式 alias 优先级高于自动展开。
- `src/config/sections/models.test.ts` — 新增 6 个测试覆盖 enabled[] 展开、capabilities 透传、未知 provider 拒绝等。14 个测试全 pass。
- `src/server/routes/enabled-models.ts` — 新增 `GET /admin/enabled-models` 路由,返回当前已启用的模型列表(把 schema 里的 undefined 字段省略)。Studio 用它读取当前状态。
- `src/server/server.ts` — 把新路由挂到 `createBuiltinRoutes`。

### Studio 侧 (`studio/`)

- 新增 `studio/src/log-stream.ts` — 进程管理模块。spawn godex 用 `stdio: ['ignore','pipe','pipe']`,stdout/stderr 实时进 1000 行 ring buffer,按 emoji/keyword 分类 level (`info/warn/error/debug`)。导出 `getGodexConfig` / `getGodexBinary` / `pushLog` / `getRecentLogs` / `subscribe` / `isGodexRunning` / `getGodexPid` / `killGodex` / `startGodex`。
- `studio/src/serve.ts` 大改:
  - 删掉旧的 `killExistingGodex` / `startNewGodex` / `applyConfig` 和 trace DB 路径逻辑(全部由 `log-stream` 接管)。
  - 新路由 `POST /api/save-config` 写 `models.enabled[]` 到 config.yaml,**不重启 godex**。
  - 新路由 `POST /api/restart` 通过 `log-stream` kill + spawn,**只重启 godex 进程**。
  - 新路由 `GET /api/logs/stream` 用 SSE 把 ring buffer 实时推到前端 (`event: line` + `event: ready`)。
  - 新路由 `GET /api/status` 一次返回 godex health + studio 进程信息 (pid / binary / config path)。
  - 新路由 `GET /api/enabled-models` 代理到 godex `/admin/enabled-models`,前端不用记两份。
  - `GET /` 服务 `public/index.html`,自动探测 `<exe>/public/index.html` 或源码目录,运行时不会因为 cwd 找不到。
  - `readAllProviders` 修复:支持 provider 名字含 `.` 或 `/`(`minnimax.chat` / `provider/v1` 等)。
- `studio/public/index.html` 全重写,两栏布局:
  - **左栏**:Provider 列表 (CRUD),点 active 高亮,× 删除带确认。
  - **右栏**:Active Provider 编辑器,3 个 section:
    - (a) 连接设置:Name / Base URL / API Key / Spec / Timeout / Note。
    - (b) Enabled Models:`Fetch from remote` 拉 `/v1/models` 批量添加,每行有 `[✓ include] [model id] [provider tag] [preset?]` + `ctx / max` 两个数字 input + 8 个 capability chips (点击切换 on/off) + remove 按钮。`+ Add manually` 弹 `prompt()` 加任意 model id。`Clear all` 一键清空。
    - (c) Action bar:`Save enabled[]` (写 yaml 不重启) / `Save & Restart godex` (写 + restart) / `Reset`。
  - 底部 200px 高 logs panel,SSE 实时刷,filter 按 source (godex / studio) + level (info / warn+ / error only)。
  - 顶部 header 显示 godex 健康 (dot + text + pid) + studio→godex 端口映射。
- 12 个常见 model id 的 preset (MiniMax-M3 / M2.7 / M2.7-highspeed / deepseek-reasoner / chat / coder / o1 / o3 / o4-mini / gpt-4o / gpt-4-turbo / glm-4-plus / glm-4-air) 自动套 context_window / max_tokens / capabilities,用户可逐项覆盖。
- `studio/tsconfig.json` 不再 `extends ../../tsconfig.json`(原路径在父目录找不到),改成自包含配置。

### 打包

- `bun build ./studio/src/serve.ts --compile --target bun-windows-x64 --outfile platforms/win32-x64/bin/studio.exe` 产出 98 MB 的单文件 exe,旁边 `public/index.html` 会被自动探测,无需 cwd 限制。
- `platforms/win32-x64/bin/public/index.html` 是 exe 运行时找的默认位置,源码改了记得同步拷贝过去。

## 行为变化

- **两栏 vs 三栏**:中间"模型参数"表单整个砍掉,参数直接 inline 在每个 enabled model 行的 input/cap chips 上,所见即所得。
- **`Save` vs `Save & Restart`**:v1 一键就 restart godex,v2 把这两个动作拆开,默认只写 yaml 不中断服务。
- **多 provider**:config.yaml 里的 `models.enabled[]` 跨所有 provider 累计,UI 一次展示所有,每个 row 标 `[provider tag]`。
- **provider 名字允许 `.` / `/`**:之前 regex 写死 `[A-Za-z0-9_-]+`,碰到 `minnimax.chat` 直接漏掉。

## 兼容性

- **godex 0.x 旧 config**:不写 `models.enabled[]` 时行为完全不变。Studio 只在自己初始化或者用户点 `Save enabled[]` 时才会往 yaml 里加 `models.enabled` 段。
- **aliases 旧写法**:仍受支持,只把 `enabled[]` 看成是 aliases 的另一种生成方式 (自动展开为 `<provider>/<model>` 自指)。
- **Studio v1 备份**:`studio/src/serve.ts.bak.v1` + `studio/public/index.html.bak.v1` 保留,rollback 只需 `cp` 回来。

## 已知限制

- Studio 当前只看一个 `config.yaml` (`GODEX_CONFIG` env 或默认 `C:\\Users\\Bliss\\.godex\\config.yaml`),不监听多 config。
- enabled[] 的 `multimodal` 字段还在 schema 里,但 UI 没暴露(v1 时代遗留,v2 暂不删,等用户实际需要再加)。
- 如果上游 `/v1/models` 超过 ~500 个 model,`Fetch from remote` 会一次性全加进 enabled[],没有分页。
- `POST /api/restart` 用 `taskkill /F` 强杀端口监听者,如果 5678 上跑的是别的进程也会一起被干掉 (没做"只杀 godex" 的精确 PID 探测)。
